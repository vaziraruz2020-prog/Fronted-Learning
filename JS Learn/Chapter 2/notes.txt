ВСЕ сущности в JS - Объекты

Массив, Число, Функция, Строка - это ОБЪЕКТЫ
НО Число и Строка только ведут себя как Объекты

Любое выражение всегда возвращает значение
|| - или в JS
<= >= - больше/меньше или равно 
!== - не равно

Переменные дают возможность повторного доступа к значениям
Название переменных должны быть понятными

      Объявление переменных
    let      const       var
let - дает возможность изменять значения переменных
const - дает одно определенное значение для одной переменной, также предотвращает возможные проблемы связанные с динамической типизацией
var - работает как "let", но объявлеяет переменные глобально т.е. если у нас разделенно на группы код 
      "let" будет работать только в определенной группе, а "var" будет работать везде


Переменная => Имеет значение => Значение имеет тип
Типы:
Примитивные типы:
string(строка), boolean(логический), number(число), symbol(символ), null, undefined

Ссылочный тип:
object(объект)

Правила работы с переменными:
1. Все переменные объявлять до их использования
2. Стараться испльзовать const везде, где это возможно

Всегда лучше использовать точечную запись чем скобочную запись
delete myCity.info['isPpular']
delete myCity.info.isPpular <= better

Cкобочная запись нужна когда мы записываем выражение например:
delete myCity.info[20 -  10]
delete myCity.info['isPpular' + 200 * 2]

Функция - это блок кода который можно выполнять многократно
Функция может быть: Имменованной; Присвоена переменной; Анонимной; Аргументом при вызове другой функции; Методом
Функция - это объект
Внутри функции не рекомендуется мутировать внешние объекты

Области видимости - определяет границы действия переменной
Правила работы с переменной:
1. Все переменные объявлять перед их использованием
2. Стараться использовать const везде, где это возможно
3. Внутри функции не изменять переменные с внешних областей видимости

Операторы - это встроенная функция
Узнаем:
1. Сами Операторы
2. Операнды
3. Унарные и Бинарные Операторы
4. Инфиксная, Префиксая и Постфиксная запись
            Операторы:
      +     -     *     /     <---- Арифметические
     ===   !==    <=    >=    <---- Сравнение
         !     &&    ||       <---- Логические
               =              <----Присваивания

       Текстовые Операторы:
      tyoeof      instaneof
      new         delete

Унарные операторы всегда имеют один операнд
Бинарные операторы имеют два операнда

В инфиксной записи оператор стоит между операндами
В префиксной записи оператор стоит перед операндом
В постфиксной записи оператор стоит за операндом

            Оператор &&
      Выражение 1 && Выражение 2
            |
      Если false то:
1. Выражение 2 игнорируется
2. Возвращается результат "Выражения 1"

            Оператор ||
      Выражение 1 || Выражение2
            |
      Если true то:
1. Выражение 2 игнорируется
2. Возвращается результат "Выражения 1"

Операторы && и || всегда должны возвращать какое-то значение не важно оно true или false 
Оператор ... нужен для разделения свойтсва объекта в другой объект,
проще говоря копировать свойства из другого объекта и вставляя эти свойства в другой объект при этом не копируя ссылку на предыдущий объект

Оператор + для конкатенации строк
'Hello ' + 'World'

Функциональные выражения всегдда анонимные

Объявленная функция:
Имеет имя: да
Можно использовать автономно: да
Можно присваивать переменной: да
Можно использовать как аргумент в вызове другой функции: да

Функциональное выражение:
Имеет имя: нет
Можно использовать автономно: нет
Можно присваивать переменной: да
Можно использовать как аргумент в вызове другой функции: да

Стрелочная функция:
Имеет имя: нет
Можно использовать автономно: нет
Можно присваивать переменной: да 
Можно использовать как аргумент в вызове другой функции: да

Стрелочные функции это выражение и они всегда анонимны

Разница между Функциональным выражениеми Стрелочной функции в том что Стрелочная функция не имеет имя и function.
Это все прописывается путем () => {}
() - параметры
=> - обзначение самого function
{} - тело функции
Желательно всегда добавлять скобки не учитывая кол-во парметров

Выражение всегда возвращает значение
Инструкция выполянет действия

Инструкция обыяно заканчивается точкой с запятой
Исключение: Блоки

Выражение модет быть инструкцией
Инструкция не можжет быть выражением
Выражения могут быть использованы как аргументы

Массивы - это объект с цифровыми именами свойтсв

Основные методы массивов - push, pop, sift, unshift, forEach, map

Условные инструкции - if, if ... else, switch, тернарный оператор
У тернарного оператора три операнда
Конструкция с тернарным оператором - выражение
Вывражение возвращает значение
